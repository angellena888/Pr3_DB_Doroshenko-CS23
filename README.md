Рис. 4 – Контекстна діаграма в середовищі Draw.io
Завдання 6. 
Використовуючи інструменти штучного інтелекту, необхідно здійснити опис побудованої контекстної діаграми, виконати її аналітичне опрацювання та включити результати до звіту. Такий підхід забезпечує розвиток навичок критичного аналізу й інтеграцію сучасних технологій у навчальний процес. 
Контекстна діаграма відображає роботу системи продажу авіаквитків як єдиної цілісної функції, що взаємодіє з зовнішнім середовищем через чотири типи інтерфейсних стрілок: вхідні дані, керуюча інформація, механізми та вихідні результати.
1. Вхідні дані (Input)
До системи надходять інформаційні потоки, які використовуються для подальшої обробки:
•	Дані про рейс – ідентифікаційна інформація, маршрут, код рейсу.
•	Відстань між містами – основний фактор для розрахунку вартості.
•	Розклад рейсів – доступні дати та часи вильоту.
•	Ціни на квиток / тарифи – базові та акційні тарифи авіакомпанії.
Ці дані є ключовим ресурсом, що визначає можливість розрахунку вартості та побудови маршрутів.
2. Керуючі впливи (Control)
Це правила та обмеження, які визначають, як саме система повинна виконувати свою роботу:
•	Розклад рейсів (в межах нормативів)
•	Типи літаків і класів обслуговування
•	Комісійні збори
•	Правила тарифікації
Контролі задають бізнес-логіку: як рахувати ціну, які рейси допустимі, які класи доступні, як застосовуються збори.
3. Механізми (Mechanism)
Ресурси, які забезпечують виконання процесу:
•	Система (База даних рейсів) – основний технічний інструмент.
•	Тарифний модуль / менеджер – програмний компонент, що виконує розрахунки.
•	Менеджер – людський ресурс, який може підтвердити операції, коригувати тарифи або обробляти нестандартні запити.
Механізми дозволяють системі функціонувати технічно та організаційно.
4. Вихідні дані (Output)
Результатом роботи системи є інформаційні продукти, необхідні клієнтам:
•	Оброблені дані про рейс
•	Список доступних рейсів
•	Відомість тарифів
•	Інформація для клієнтів (маршрути, ціни, умови, час вильоту тощо)
Ці дані є кінцевими результатами, які система формує для користувачів або зовнішніх систем.
Аналітичне опрацювання діаграми
Контекстна діаграма моделює систему продажу авіаквитків на найвищому рівні абстракції. Вона дозволяє визначити чіткі межі системи та зрозуміти, які зовнішні об'єкти впливають на її роботу.
Аналітичні висновки:
1.	Система залежить від великої кількості вхідних даних, зокрема від тарифів, відстані та структурованої інформації про рейси. Це означає, що будь-яка помилка в цих вхідних даних може призвести до неправильного формування ціни або маршруту.
2.	Керуюча інформація відіграє ключову роль, оскільки правила тарифікації, типи літаків і комісійні збори визначають реальні бізнес-процеси авіакомпанії. Зміни у цих правилах впливають на всю роботу системи.
3.	Механізми показують, що система функціонує на основі комбінації програмних модулів і людського фактору. Це вказує на те, що автоматизація процесу неповна, і певні дії можуть залежати від менеджерів.
4.	Вихідні результати системи прямо відповідають запитам клієнтів.
Сформована інформація (список рейсів, ціна, тарифні умови) дозволяє користувачу приймати рішення щодо покупки квитка.
5.	Діаграма демонструє логічну структуру процесу, де:
o	Input → створює дані
o	Control → визначає правила
o	Mechanism → виконує операції
o	Output → повертає результат
6.	Модель дозволяє визначити вузькі місця системи, наприклад:
o	залежність від точності БД,
o	складні тарифні правила,
o	можливі затримки через участь менеджера.
7.	Контекстна діаграма є основою для подальшої декомпозиції, де процес можна розділити на:
o	A1 — Обробка рейсів
o	A2 — Розрахунок вартості
o	A3 — Обробка запитів клієнтів
Таким чином, побудована модель створює цілісне уявлення про те, як працює система продажу авіаквитків, і забезпечує базу для деталізації та оптимізації процесу.
